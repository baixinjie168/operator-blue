# basic_info 校验规则

本文件定义了从 CANN 算子说明文档中校验 **basic_info（基本信息）** 的严格规则。

## 校验字段

文档必须包含以下字段：

- operation_name(算子名称)
- description(算子描述)
- api_flow(阶段步骤)

## 校验规则

### 1. operation_name(算子名称)

- 来自文档标题或第一行的算子名称。

### 2. description(算子描述)

- 从"功能说明"章节校验算子功能描述。
- `description` 只允许承载“算子功能/接口功能”本身，不允许混入计算公式、参数解释、调用流程或其他模块内容。
- 校验 `description` 时必须回到文档“功能说明/接口功能”原文核对；如果结果混入 `out_i = ...` 等计算公式，应判定为业务错误。
- 如果文档把功能说明和公式写在同一段，应校验结果是否只准确截取了功能语义，而不是整段照搬。
- 如果结果把参数取值范围、示例 shape、调用步骤或参数解释拼进 `description`，即使语句通顺，也应判定为字段职责错误。
- 变量名、下划线转义、大小写及原文占位符必须与文档保持一致。
- 如果文档没有稳定的功能说明句，应接受保守提取，但不接受凭空补写原文没有直接给出的信息。
- 如果文档功能说明较短，不能因结果更“完整”就放宽校验；凡是文档未直接给出的推断描述，都应提示删除或收敛。

### 3. api_flow(阶段步骤)

从"函数原型"章节判断 API 流程类型：

- **two-step**：存在两段式接口（先调用 GetWorkspaceSize 接口，再调用实际执行接口）
- **one-step**：单段接口，直接调用即可
- `api_flow` 必须只依据函数原型和调用流程判断，不接受从描述性文字臆测阶段数。
