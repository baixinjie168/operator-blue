# basic_info 校验规则

本文件定义了从 CANN 算子说明文档中校验 **basic_info（基本信息）** 的严格规则。

## 校验字段

文档必须包含以下字段：

- operation_name(算子名称)
- description(算子描述)
- api_flow(阶段步骤)

## 校验规则

### 1. operation_name(算子名称)

- 来自文档标题或第一行的算子名称。

### 2. description(算子描述)

**提取来源校验**：
- 必须从”功能说明”章节提取算子功能描述，优先抽取明确对应”算子功能”或”接口功能”的句子。

**内容范围校验**：
- `description` 只允许承载”算子功能/接口功能”本身，不允许混入计算公式、参数解释、调用流程、参数约束、平台支持等非功能信息。
- 如果文档把功能说明和公式写在同一段，应校验结果是否只准确截取了功能语义，而不是整段照搬。

**文本处理校验**：
- 变量名、下划线转义、大小写及原文占位符必须与文档保持一致。

**保守原则校验**：
- 如果文档没有稳定的功能说明句，应接受保守提取，但不接受凭空补写原文没有直接给出的信息。

**示例要求**：
- 应类似以下格式：
  - “算子功能：输入为BOOL型tensor时，进行逻辑非运算；输入为INT型时进行按位非运算。”
  - “接口功能：从start起始到end结束按照step的间隔获取值，并保存到输出的1维张量，其中数据范围为[start,end)。”
  - “将输入的所有元素限制在[min, inf]范围内。”

### 3. api_flow(阶段步骤)

从"函数原型"章节判断 API 流程类型：

- **two-step**：存在两段式接口（先调用 GetWorkspaceSize 接口，再调用实际执行接口）
- **one-step**：单段接口，直接调用即可
- `api_flow` 必须只依据函数原型和调用流程判断，不接受从描述性文字臆测阶段数。
